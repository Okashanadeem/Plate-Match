# PlateMatch Project Constitution

## Core Mandates

1. **Strategic Planning First:** Before starting any phase, I MUST read the corresponding `plan.md`, `task.md`, and `implementation.md` files thoroughly to ensure full alignment with the project's goals.
2. **Task Tracking:** Every task completed MUST be immediately marked in the `task.md` file of the current phase.
3. **Progress Updates:** The central `specs/README.md` must be updated as each phase progresses or is completed.
4. **Surgical Implementation:** Changes should be precise and strictly follow the documented implementation strategy for each phase.
5. **No Data Modification:** The original dataset folder must NEVER be modified. All processing must be client-side and output only via the Export Engine.
6. **Focus on Speed:** The tool is designed for human efficiency. Keyboard shortcuts and near-instant transitions are non-negotiable performance requirements.
7. **Maintain Clean Code:** Adhere to TypeScript and Next.js best practices, ensuring type safety and modular component design throughout the codebase.

## Execution Rules

- **Read twice, implement once:** Ensure complete understanding of the phase requirements before generating code.
- **Incremental Validation:** Verify each task's outcome before moving to the next.
- **Error Handling:** Graceful failure is expected; silent crashes are not acceptable.
